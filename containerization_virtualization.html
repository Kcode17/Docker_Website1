Containerization vs Virtualization

How do they compare?

   

Containers

 

Containers are an abstraction at app level where all the dependent libraries and binaries are bundled together. Multiple containers can run on a same machine as they share the host OS and each container running its individual & isolated processes.

 

Virtual Machines

 

Virtual Machines are an abstraction of utilizing one server as many servers. Each Virtual machine comprises of a full operating system, the necessary application along with binaries and libraries.

 

Comparing Containerization and Virtualization

 

Both the technologies have very similar goals to distribute application software in isolated packages.The difference comes in the approach and level of virtualization.

  • Containers running on the host OS just need the necessary libraries and binaries, which makes them light weight and faster to boot up.
  • Containers have better resource utilization compared to virtual machines.
  • Containers require less memory space (MBs) compared to Virtual Machines which require a lot of space (GBs) for operating system.
